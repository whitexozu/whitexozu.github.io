    [Ubuntu 16] Initial install package + nginx + gunicorn + flask

    ubuntu 16을 개인 노트북에 설치 후 ssh와 mysql 설치, 그리고 nginx + gunicorn + flask 조합으로 웹서비스를 구성 해보겠습니다.

    openssh-server

    1. 설치

       $ sudo apt-get update
 $ sudo apt-get upgrade
 $ sudo apt-get install openssh-server

    2. 만약 의존성 실패가 나오면

      • cd /etc/apt
      • sudo vi source.list
      • :%s/kr.archive.ubuntu.com/ftp.daumkakao.com
      • :wq
      • sudo apt-add-repository “deb http://archive.ubuntu.com/ubuntu precise main restricted”
      • sudo apt-get update
      • sudo apt-get upgrade
      • sudo apt-get install openssh-client=1:7.2p2 로 강제 버전 지정
      • sudo apt-get install openssh-server

    노트북 전원 유지

    1. 설정

       $ sudo vi /etc/systemd/logind.conf

       ...
 #HandleLidSwitch=suspend
 # 위 내용을 아래 내용으로 변경
 HandleLidSwitch=ignore
 ...

    2. 서비스 재실행

       $ systemctl restart systemd-logind

    git 설치

    1. 설치 
       $ sudo apt-get install git

    curl 설치

    1. 설치 
       $ sudo apt install curl
    2. 의존성 실패가 나오면 
       $ sudo apt install curl

    nvm 설치

    1. 설치 
       $ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.34.0/install.sh | bash
    2. 명령어 참고
      • nvm ls-remote : nvm이 지원하는 버전 확인
      • nvm install [버전] : nvm에서 지원 버전 install
        • ex) $ nvm install v13
      • nvm ls => 설치된 node 버전 확인
      • nvm use v12.18.2 => v12.18.2 버전의 node를 사용
      • nvm alias default [버전] => default 버전을 변경 및 유지

    python3.6

    1. 설치

       $ sudo add-apt-repository ppa:deadsnakes/ppa
 $ sudo apt-get update
 $ sudo apt-get install python3.6
 $ sudo apt-get install python3.6-gdbm

    2. 파이선 버젼 변경 등록

       $ sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.5 1
 $ sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.6 2

    3. 파이선 버젼 변경

       $ sudo update-alternatives --config python3 // python 버전 변경

    4. 확인

       $ cd /usr/bin
 $ ll | grep python3

    pip3

    1. 설치

       $ sudo apt-get install python3-pip
 $ python3 -m pip install --upgrade pip

    2. 확인

       $ cd /usr/bin
 $ ll | gerp pip3

    mysql 8

    1. 설치

       $ mkdir ~/download
 $ cd ~/download
 $ wget https://dev.mysql.com/get/mysql-apt-config_0.8.12-1_all.deb
 $ sudo dpkg -i mysql-apt-config_0.8.12-1_all.deb
     > MySQL Server & Cluster 선택
     > MySQL 8 선택
     > OK 선택
 $ sudo apt update
 $ sudo apt install mysql-server
     > root 비밀번호 입력
     > root 비밀번호 재입력
     > Use Strong Pasword Encryption 암호방식 선택
 $ mysql --version
 $ sudo service mysql start
 $ mysql -u root -p

    2. db 생성

       mysql> create database nlp;
 mysql> create user 'nlp_admin'@'%' identified by 'p@ssWord';
 mysql> grant all on nlp.* to 'nlp_admin'@'%';
 mysql> flush privileges;

    3. 삭제

       $ sudo service mysql stop

 $ sudo apt-get remove --purge mysql-server -y
 $ sudo apt-get remove --purge mysql-server* -y
 $ sudo apt-get remove --purge mysql-common -y
 $ sudo apt-get autoremove

 $ sudo rm -rf /var/log/mysql
 $ sudo rm -rf /var/log/mysql.*
 $ sudo rm -rf /var/lib/mysql
 $ sudo rm -rf /etc/mysql

    flask

    1. 설치

       $ pip3 install Flask==1.1.2

    2. flask app 생성

       $ mkdir ~/dev
 $ mkdir ~/dev/project
 $ mkdir ~/dev/project/thesaurus
 $ vi ~/dev/project/thesaurus/app.py

       from flask import *
 app = Flask(__name__)

 @app.route("/")
 def hello():
     return "<h1 style='color:blue'>Hello Thesaurus</h1>"

 if __name__ == "__main__":
     app.run(host='0.0.0.0', port=5000, debug=True)

    3. 실행

       $ python3 ~/dev/project/thesaurus/app.py

    4. 테스트

      • 해당 서버 IP:5000로 브라우저 접속 테스트
      • ctrl + c 로 종료

    gunicorn

    1. 설치 
       $ pip3 install gunicorn

    2. flask 연결 설정

       $ vi ~/dev/project/thesaurus/wsgi.py

       from app import app

 if __name__ == "__main__":
     app.run()

    3. 테스트

       $ cd ~/dev/project/thesaurus/
 $ gunicorn --bind 0.0.0.0:5000 wsgi:app
      • 해당 서버 IP:5000로 브라우저 접속 테스트
      • ctrl + c 로 종료

    4. 데몬 서비스 등록

       sudo vi /etc/systemd/system/thesaurus.service

       [Unit]
 Description=Gunicorn instance to serve thesaurus
 After=network.target

 [Service]
 User=sin
 Group=www-data
 WorkingDirectory=/home/sin/dev/project/thesaurus
 ExecStart=/home/igs_admin/.local/bin/gunicorn --workers 1 --timeout 300 --bind unix:thesaurus.sock --error-logfile /var/log/gunicorn/error.log --access-logfile /var/log/gunicorn/access.log --capture-output --log-level debug -m 007 wsgi:app
 Environment=PYTHONUNBUFFERED=1

 [Install]
 WantedBy=multi-user.target

      ExecStart의 gunicorn 경로는 which 명령으로 확인

    5. 로그 폴더 생성

       $ sudo mkdir /var/log/gunicorn
 $ sudo chmod 777 /var/log/gunicorn

    6. 서비스 실행

       $ sudo systemctl start thesaurus
 $ sudo systemctl enable thesaurus
 $ sudo systemctl status thesaurus
    7. 확인 
       $ ll /home/sin/dev/project/thesaurus/thesaurus.sock

    nginx

    1. 설치

       $ sudo apt-get install nginx
 $ nginx -v

    2. 설치 확인

      • 해당 서버 IP로 브라우저 접속 테스트

    3. sites-available에 wsgi 등록

       sudo vi /etc/nginx/sites-available/thesaurus

       server {
     listen 80;
     server_name localhost;

     location / {
         include proxy_params;
         proxy_pass http://unix:/home/sin/dev/project/thesaurus/thesaurus.sock;
     }
 }

    4. sites-enabled에 링크 생성

       sudo ln -s /etc/nginx/sites-available/thesaurus /etc/nginx/sites-enabled

    5. nginx 테스트

       sudo nginx -t

    6. nginx reload

       sudo nginx -s reload

    7. 테스트

      • 해당 서버 IP로 브라우저 접속 테스트
